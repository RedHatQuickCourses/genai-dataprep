Docling Web UI

The docling-serve project provides an easy-to-use web-based UI to experiment and prototype document conversion using Docling.

The most convenient way to run it is as a container.

Do not expose the web UI to the internet or external use unless you know what you are doing. Ideally, you must provision a reverse proxy, load balancer, or an API gateway in front of docling-serve for security reasons. It is mainly meant to be used to test, experiment, and prototype document parsing and conversion.

Pre-requisites

  • You must have installed Podman or Docker on your system

Lab: Using the docling-serve Web UI

  1. Run the docling-serve container using Podman

    $ podman run -p 5001:5001 \
  -e DOCLING_SERVE_ENABLE_UI=true \

  2. Using a web browser, navigate to http://localhost:5001/ui/ to load the Docling server web UI

    Figure 1. Docling serve web UI

  3. The web UI provides two tabs for a URL containing an input document (Convert URL) and another for local files that can be uploaded from the system (Convert File). Enter https://arxiv.org/pdf/2408.09869 in the URL Input Source field. In the Options section, select the Markdown option and unselect JSON. Leave all other fields at the default values.

  4. Click Process URL to convert the document. Click the Markdown tab in the output section.

    Figure 2. Markdown output in web UI

  5. You can click the Markdown-Rendered tab in the output section to preview how the Markdown is rendered.

    Figure 3. Markdown output preview in web UI

  6. Click the Convert File tab at the top of the web UI and upload a sample input document from the sample-data folder in the genai-apps Git repository, or some other file in one of the supported formats from your local file system. Click Process URL to convert the document, and verify the output.

  7. Experiment with different combinations of input and output formats and check the output.

  8. docling-serve also provides a REST API that you can use to integrate with third-party client applications. Navigate to http://localhost:5001/docs to view the OpenAPI (Swagger) API documentation for docling-serve.

    Figure 4. Docling serve REST API

  9. Send an HTTP request using the curl tool to test the REST API by converting a document (hosted at an external URL). You should see a JSON response as output of the conversion. The Markdown content is under the md_content attribute of the JSON response.

    $ curl -X 'POST' \
  'http://localhost:5001/v1alpha/convert/source' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "http_sources": [{"url": "https://arxiv.org/pdf/2501.17887"}]
  }'
